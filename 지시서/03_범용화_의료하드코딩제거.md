# P3 — 범용화: 의료 하드코딩 제거 → 전체 한국법

## 목적
lawbot은 현재 여러 곳이 의료 도메인에 고정돼 있다. 범용 법률 제품이 되도록 의료 한정 로직·문구를 제거하고, "주 근거 + 참고 인접법" 등급 분리를 추가한다.

## 대상 파일 (현재 코드 확인 후)
- `search/rag.py` — 시스템 프롬프트가 "의료법 전문가"로 고정, 의료/의약품 외 질의 거부 로직 존재.
- `search/ad_review.py` — 의료 고정의 **실제 위치는 판정 시스템 프롬프트(`_JUDGE_SYSTEM`, 의료법 제56조 체크리스트)와 `flt={"doc_type":"law"}` 필터**. ⚠️ `AD_LAW_TITLES` 상수는 정의돼 있으나 **실제 스코프에 안 쓰이는 죽은 상수** — 거길 고쳐도 효과 없음. 프롬프트와 필터를 고칠 것.
- `api/main.py` — OpenAPI DESCRIPTION "의료 코퍼스" 문구.
- `config.py` — `ANSWER_DISCLAIMER` 등 의료/자치법규 문구.

## 작업 단계
1. **RAG 프롬프트 일반화** (`search/rag.py`):
   - "의료법 전문가" → **도메인 무관 한국법 전문가(전문가 모드)**.
   - 의료/의약품 외 질의를 거부하는 도메인 게이트 제거 또는 "범용 허용"으로 변경. 단 **비법률 잡담은 친절 안내**(버그/UX 항목, P6와 연동): "한국 법령·판례 기반 법률 질문에 답합니다."
   - 근거 없으면 침묵·확인필요 원칙은 유지.
2. **주 근거 / 참고 인접법 등급 분리** (범용판 "놓칠 수 있는 부분"):
   - 답변 구조를 `주 근거법령 → 결론 → ⚠️ 함께 검토할 인접법(트리거 충족 시만)` 2단으로.
   - 인접법은 **트리거 기반**(구성요건이 실제로 걸릴 때만) — 키워드만 겹쳐 끌어오는 과잉 인용 금지(전문가의견 문서 원칙).
   - Structured Outputs 스키마에 `primary_citations`/`related_citations` 분리 필드 추가, 둘 다 사후검증.
3. **ad_review scope 범용화** (선택):
   - 광고검토를 의료 외 일반 광고/표시로 확장하거나, 광고검토는 한 버티컬로 두고 일반 질의는 `/v1/ask`로. 제품 결정에 따름(README 2장: 범용).
   - 최소한 buggy한 `doc_type=law` 고정(버그 #6)은 행정규칙 포함으로 수정.
4. **문구 정리**: API DESCRIPTION·disclaimer를 범용 한국법으로. "전국 자치법규" 거짓 커버리지 문구 제거(조례 0건이므로) 또는 "조례 별도 제공 예정".

## 완료 기준 (DoD)
- 비의료 질의(예: "근로기준법 연차휴가", "전세 계약 갱신")가 거부되지 않고 근거 단 답을 반환.
- 답변에 주 근거/참고 인접법이 분리되어 나오고, 인접법은 트리거 시에만 등장.
- 비법률 입력("안녕")에 딱딱한 거부 대신 안내 문구.
- `python -m search.rag` 관련 셀프체크/기존 테스트 회귀 없음.

## 진행 상황 (2026-06-29)
- ✅ **rag.py 범용화**: SYSTEM_PROMPT를 "MediLaw 의료법 전문가"→**"lawbot 한국 법령·판례 전반"**으로. 도메인 가드는 의료-한정 제거, 비법률 잡담(인사·코딩 등)만 안내문구로 막음(grounding·인용·불확실성 규칙 전부 유지). 코드 레벨 의료 게이트 없음 확인.
- ✅ **api/main.py**: 상단 DESCRIPTION·커버리지 문구 범용화("한국 법령·판례 RAG +의료광고 검토"). ad-review 태그는 실제 의료광고 버티컬이라 유지(정확).
- ✅ **config.ANSWER_DISCLAIMER**: 거짓 "전국 자치법규" 커버리지 제거 → 실제(법령·행정규칙·판례, 자치법규 제외)로 수정.
- ✅ **기능 테스트**: 비의료 질의(근로기준법 연차)가 **거부되지 않고** 처리됨(거부문구 False). 단 해당 질의는 검색이 판례만 올려 "근거 불충분"으로 응답(firewall 정상 — 환각 안 함). → **"법명+주제 질의 시 통치 조문(doc_type=law) 우선 부스트"가 남은 검색품질 과제**(cite-001과 동일). P5/P2 후속.
- ◻️ 주근거/참고 인접법 등급 분리(P3 step2)는 미구현(선택).

## 환각 방지 체크
- 프롬프트에서 도메인 제한을 풀 때 **근거검증 게이트는 절대 풀지 말 것**(범용≠환각 허용).
- 인접법 자동안내가 과잉되지 않게 트리거 조건을 코드로 명시.
- 실제 거부 로직 위치를 코드에서 확인 후 수정(추정 금지).
