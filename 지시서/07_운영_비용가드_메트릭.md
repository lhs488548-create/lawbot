# P7 — 운영: 런타임 비용 가드 · 외부호출 안정화 · 메트릭

## 목적
서비스 등급 안정성을 위한 런타임 가드. 한 사용자가 비용을 폭주시키지 못하게 하고, 외부 호출(law.go.kr)을 안정화하며, 운영 지표를 본다.

## 대상 파일
- `api/auth.py`, `api/keys.py`, `api/db.py` — 키별 쿼터.
- `api/main.py` — 레이트리밋(slowapi 이미 사용), 데모 IP 리밋.
- `search/ad_review.py`, `search/verify.py` — law.go.kr 호출부.
- (신규) `api/metrics.py` — 지표 수집.

## 작업 단계
1. **키별/IP별 쿼터**:
   - 발급 키: 일일 토큰 상한·호출 쿼터. 초과 시 429 + 친절 메시지.
   - 데모(무인증, P6): IP당 분당 N회.
   - 특히 `/v1/ad-review`는 1건이 (주장 수 × (임베딩+검색+law.go.kr 최대 12회))를 태우므로 상한 필수.
2. **외부호출 안정화** (law.go.kr):
   - ✅ **이미 있음**(재작업 금지): 타임아웃(`search/verify.py` `_HTTP_TIMEOUT=12.0`) + tenacity 재시도(백오프).
   - ➕ **신규 필요**: **캐시 + 서킷브레이커**만. (현재 캐시·CB 없음.)
   - 캐시 키 = 법령/조문 + 기준일(현행성 깨지지 않게 기준일 포함). TTL 적정값.
3. **메트릭**:
   - 게이트 발동률, 인용 폐기율, as_of_date 미적용 비율, 임베딩 캐시 히트율, 쿼리 지연(p50/p95), 429율, 에러율.
   - 최소 구조적 로깅 → 후속에 Prometheus/대시보드 가능하게.
4. **런타임 비용 로깅**: 일일 OpenAI 토큰/비용 누적, 상한 초과 시 경고/자동 차단.
   - ⚠️ `api/auth.record_tokens`와 `ApiKey.tokens` 컬럼이 **이미 있으나 어디서도 호출되지 않음** → 새 미터 만들지 말고 비용 엔드포인트(`/v1/ask`·`/v1/ad-review`)에서 `record_tokens` 호출로 배선.

## 완료 기준 (DoD)
- 데모 IP 리밋·키 쿼터 동작(초과 시 429 + 메시지).
- law.go.kr 호출에 타임아웃/캐시/CB 적용, 외부 장애 시 전체 요청이 안 멈춤.
- 핵심 지표가 로그/엔드포인트로 확인됨.

## 환각 방지 체크
- slowapi·auth의 현재 구현을 확인 후 확장(중복 구현 금지).
- 캐시가 현행성(as_of_date)을 깨지 않게 키에 기준일 포함.
