# 산출물 5 — 데이터 갭 분석 및 추가 수집 계획

> **목적**: 현재 확보된 6개 ZIP 코퍼스(약 1,394만 항목, 9.3GB, 전부 공공저작물 — 저작권법 §7)를 진단하여 **무엇이 빠졌는지**를 표로 정리하고, 각 갭의 **중요도(필수/권장/선택)와 영향**을 평가한다. 그다음 "**단순 API로 끝낼 경우 필요 데이터**"와 "**고도화 lawbot에 필요한 전국·전량 데이터**"를 구분하여, 누락 데이터를 **어디서·어떻게** 수집할지(인증키 신청·요청형식·예상 규모·라이선스) 구체적 계획·우선순위·체크리스트로 제시한다.
>
> **데이터 출처 표기**: 본 문서의 코퍼스 수치는 전부 `D:/법파일/_samples/corpus_stats.txt`(직접 Read 검증) 기준이며, 외부 주장에는 출처 URL을 병기한다.

## 0. 요약 (Executive Summary)

현재 코퍼스는 **판례 + 4개 지자체 자치법규 + 국가 행정규칙**으로 구성된, 양적으로는 두텁지만 **법체계의 1차 근간(국가 법령 본문)이 통째로 비어 있는** 편향된 데이터셋이다.

| 핵심 진단 | 수치/내용 | 출처 |
|---|---|---|
| 전체 ZIP 총 항목수 | **13,938,996** (약 1,394만), 9.3GB | corpus_stats.txt 합산 |
| 인덱싱 대상 현행 청크 | 판례 123,743 + 현행 조문 3,287,831 = **약 341만** | corpus_stats.txt |
| 과거버전 스냅샷 비중 | 전체 항목의 약 76%가 비-current(과거버전·중복) | 파생 계산 |
| **치명적 갭 1** | 국가 1차 법령(법률·시행령·시행규칙) 본문 **0건** | phase2 |

| Q5 | **비식별화(가명) 35%** — 원고 ○○○ 등 | 인물 기반 질의 불가, 임베딩 노이즈 | [PERSON_n] 정규화, 가드레일 명시 |

| Q7 | **ID 체계 이질성** — 자치법규(숫자) vs 행정규칙(`ADMRULE:n`) vs 판례(precSeq) | 통합 doc_id 충돌 | 네임스페이스 prefix 정규화 |

> **데이터 정체성 경고(필수 인지)**: 본 코퍼스는 명칭과 달리 "한국 법률 전체"가 아니라 **4개 광역 자치법규 + 국가 행정규칙 + 판례**다. 서비스명을 "한국 법률 RAG"로 내세우면 사용자 기대(민법·형법 등)와 심각한 불일치가 발생한다. **국가 법령 본문(G1)과 시도 확장(G2)을 채우기 전까지는 "지방 자치법규·행정규칙·판례 검색"으로 정직하게 포지셔닝**할 것. (근거: phase2 verdicts에서 반복 지적된 결함)

## 3. 수집 목표 2단계 구분: 단순 API vs 고도화 lawbot

수집 범위는 목표 제품에 따라 극단적으로 다르다. 무작정 전량을 모으면 비용·시간이 낭비되므로 **2개 시나리오로 명확히 분리**한다.

### 3.1 시나리오 A — "단순 API로 끝낼 경우" (MVP / PoC, 최소 필요 데이터)

**제품 정의**: 현재 보유 데이터 + **국가 법령 본문**만 추가하여 "법령·판례 조회 + 인용 API"를 제공. 전국 전수성·최신성은 보장하지 않고, "검색·인용·정보제공"으로 한정(변호사법 리스크 회피).

| 추가 필요 데이터 | 이유 | 규모(추정) |
|---|---|---|
| **G1 국가 법령 본문** (현행만) | 모법 없이는 자치법규/행정규칙 답변의 근거가 끊김 — **이것만은 필수** | 법률+시행령+시행규칙 약 5,000여 법령 / 조문 수십만 |
| 현재 보유분 정제 | current만 필터, 조문 재병합, 참조 정규화 | 약 341만 청크 |

- **추가로 모으지 않아도 되는 것**: 헌재·해석례·행정심판·영문·전국 자치법규·과거버전.
- **출처/방법**: `github.com/legalize-kr/legalize-kr` 저장소 `git clone`(가장 빠름) → 본 코퍼스와 동일 포맷(Markdown+YAML, `current.articles` 조문 분할)이라 파이프라인 재사용. 신선도가 필요하면 law.go.kr OpenAPI `target=law` 증분. (출처: https://github.com/legalize-kr/legalize-kr — 법령 83,874 커밋)

### 3.2 시나리오 B — "고도화 lawbot" (전국·전량 데이터)

**제품 정의**: 전국 17개 시도 자치법규 + 국가 법령 + 헌재 + 해석례 + 행정심판까지 포괄하는, 신선도가 보장되는 통합 법률 RAG.

| 추가 필요 데이터 | 중요도 | 규모(추정) | 출처 |
|---|:---:|---|---|
| G1 국가 법령 본문(현행+연혁) | 필수 | 조문 수십만~백만대 | law.go.kr `target=law` / legalize-kr |
| G2 자치법규 전국 17개 시도 + 기초 | 필수 | 현행 조문 수백만(4개 시도 외 13개 추가 시 2~3배) | law.go.kr `target=ordin` / ordinance-kr |
| G3 헌재 결정례 전량 | 권장 | 수만 건 | law.go.kr `target=detc` |
| G4 법령해석례 전량 | 권장 | 수만~십수만 건 | law.go.kr `target=expc` |
| G5 행정심판 재결례 | 권장 | 수만 건 | data.go.kr 3084452 |
| G6 별표·별지 서식 본문 | 권장 | 라벨 기준 수만~수십만 | law.go.kr 별표서식 |
| G8 증분 갱신 파이프라인 | 필수(운영) | 상시 | legalize-pipeline + LAW_OC |
| G9 판례 전량·최신화 | 권장 | 공개분 전량 | law.go.kr `target=prec` |
| G10 영문 법령 | 선택 | 법률 1,566 / 시행령 1,312(총 2,965) | KLRI elaw.klri.re.kr |

> **참고 — 전국 자치법규 규모 외삽**: 4개 시도(인구·지자체 수 하위권)가 현행 조문 약 198만(광주 272,687 + 세종 72,841 + 전남 953,907 + 전북 678,513 = 1,977,948)이다. 서울·경기 등 대규모 시도를 포함한 17개 시도 전량은 이보다 **2~3배 이상**일 것으로 추정된다. [추정]

## 4. 출처별 수집 방법 상세

각 누락 데이터를 **어디서·어떻게** 모을지 구체적으로 정리한다.

### 4.1 ★ 1차 핵심 — 법제처 국가법령정보 공동활용 Open API (open.law.go.kr)

대부분의 갭(G1, G2, G3, G4, G6, G9, G11)을 **단일 인증키(OC)**로 해결하는 최우선 소스다. (출처: https://open.law.go.kr/LSO/openApi/guideList.do)

#### (a) 인증키(OC) 신청 절차

```
1. open.law.go.kr 회원가입 (https://open.law.go.kr/LSO/usrJoin.do)
2. 활용할 데이터별 '공동활용 신청' → 담당자 검토 → 승인(통상 1~2일)
3. 마이페이지 'API인증키관리'에서 OC(=신청 ID) 확인
4. 문의: 법제처 044-200-6786 / 02-2109-6446
```
(출처: https://open.law.go.kr/LSO/information/guide.do)

#### (b) 요청 형식 (공통)

| 구분 | URL | 핵심 파라미터 |
|---|---|---|
| 목록 조회 | `http://www.law.go.kr/DRF/lawSearch.do` | `OC`(인증키), `target`, `type`(XML/JSON/HTML), `query`, `display`(최대 100), `page` |
| 본문 조회 | `http://www.law.go.kr/DRF/lawService.do` | `OC`, `target`, `ID`, `type` |

**target 코드별 매핑**

| target | 데이터 | 해당 갭 |
|---|---|---|
| `law` | 현행 법령(법률·시행령·시행규칙) | **G1** |
| `ordin` | 자치법규 | **G2** |
| `prec` | 판례 | G9 |
| `detc` | 헌재 결정례 | **G3** |
| `expc` | 법령해석례 | **G4** |
| `admrul` | 행정규칙 | (보유) |
| `lstrm` | 법령용어 | G11 |
| `trty` | 조약 | G11 |

**판례 본문 조회 예시** (출처: https://open.law.go.kr/LSO/openApi/guideResult.do?htmlName=precInfoGuide)
```
GET http://www.law.go.kr/DRF/lawService.do?target=prec&OC={인증키}&ID={판례일련번호}&type=JSON
# 단, 국세청 판례 본문은 HTML만 제공
```

**국가 법령(G1) 수집 흐름 예시**
```
# 1단계: 법령 목록
GET .../lawSearch.do?OC={키}&target=law&type=JSON&display=100&page=1
# 2단계: 각 법령ID로 본문(조문 포함)
GET .../lawService.do?OC={키}&target=law&ID={법령ID}&type=JSON
```

#### (c) 트래픽 제한 / 라이선스

| 항목 | 내용 |
|---|---|
| 개발계정 | **일 10,000건** 제한 |
| 운영계정 | 활용사례 등록 시 증설 신청 가능 |
| 라이선스 | 공공저작물 — **영리 목적 포함 자유 활용**, 단 출처표시(국가법령정보센터) 의무, 위·변조 금지 |

(출처: https://open.law.go.kr/LSO/information/service.do)

> **수집 전략**: 일 10,000건 한도를 고려해 `display=100`·`page` 순회로 **목록→본문 2단계** 수집. 국가 법령 본문(G1)은 운영계정 승인 후 야간 배치로 며칠에 걸쳐 수집하거나, **legalize-kr 직접 clone으로 우회**(아래 4.4).

> **최종 권고**: **G1(국가 법령) + G8(갱신 파이프라인)은 타협 불가의 필수**다. G1은 `legalize-kr/legalize-kr` clone으로 거의 무비용·즉시 해결되므로, **가장 적은 노력으로 가장 큰 갭을 메우는 1순위 작업**이다. G2(시도 확장)도 `ordinance-kr` clone으로 동일하게 저비용 해결 가능하다. 헌재·해석례(G3·G4)는 law.go.kr 단일 OC로 API 수집하고, 별표 본문(G6)·영문(G10)은 후순위로 둔다.
