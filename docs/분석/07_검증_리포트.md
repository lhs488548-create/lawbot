## 1. ① 1차 전수점검 의견

각 문서를 감사 에이전트가 읽고 **검증 가능한 사실 주장**을 모두 추출, 위험도(low/medium/high)로 분류했다.

- **저위험(low)** — 자명한 도메인 상식. 예: "법률 챗봇은 RAG가 사실상 표준", "저작권법 제7조는 판결 등을 보호받지 못하는 저작물로 규정". → 즉시 통과(7건).
- **중위험(medium)** — 코퍼스 통계 핵심 수치(대부분 corpus_stats와 일치하나 추적), GitHub 스타 수·창업연도 등 시점 가변 외부 사실, arXiv ID 등.
- **고위험(high)** — 상용 서비스 가격·매출·투자 수치, 모델 성능 수치(nDCG·F1·지연시간), 법정형·시행일, 클라우드 GPU 시세 등 **틀리면 치명적이고 최신성에 민감한** 주장.

1차에서 high로 분류돼 정밀검증이 필요하다고 본 대표 항목: 슈퍼로이어/빅케이스/엘박스 수치, KURE-v1·리랭커 성능, Pinecone·RunPod 가격, 변호사법 법정형·AI기본법 과태료, KBL/LegalBench-RAG arXiv 식별자, 행정규칙 과편화율·결측률, 비식별화 비율, KLRI 영문·AI Hub 데이터셋 규모.

## 2. ② 2차 정밀검증으로 넘긴 자료/텍스트 (요약)

2차로 넘긴 99건은 각각 **독립 검증관**이 (a) 코퍼스 통계 주장 → `corpus_stats.txt`·원본 zip 직접 대조, (b) 외부 사실 → 공식 1차 출처(모델카드·논문 원문·언론·공식 가격페이지) 웹 교차확인으로 판정했다.

**2차 confirmed(81건) 중 대표 — 그대로 유지:**
- 코퍼스 통계 전부: 판례 123,743(대법원 68,175·하급심 55,566), 자치법규 36,298법령·약198만 조문, 행정규칙 21,052·약131만 조문, 전체 항목 13,938,996 → `corpus_stats.txt`와 **정확히 일치**.

- 판례 섹션 보유율 판시사항 62.0%·참조조문 61.3%·판결요지 55.8%·참조판례 34.8% → 전수 census(61.7/61.0/55.5/34.7%)로 **확인**.

**2차 미통과 → 3차로 넘긴 18건** (refuted 또는 needs_correction): 아래 §6 수정 목록과 동일. 2차에서 가장 의심된 유형은 "수치 자체는 맞지만 **분모/귀속/범주**가 틀린" 경우와 "출처 없는 **추정치를 사실처럼** 단정한" 경우.

## 3. ③ 3차 최종검증

3차는 2차와 **다른 출처·다른 각도**로, 기본 입장을 "의심"으로 두고 반증을 적극 시도했다.

- **합산·산술 오류(57,350; 30,205×5)**: PowerShell·원문 재계산으로 명확히 반증 → 정정 확정.
- **귀속 오류(고시 27%, KURE 법률 nDCG, 법률 SOTA)**: 원본 zip 전수 재분석(결측 5,676건의 ord_kind 전수 분류) 및 공식 모델카드 재확인으로, "전체 대비 비율"·"평균값"임을 확정 → 표현 정정.
- **추정치를 사실로 단정(GPU 50~100ms, Pinecone $3,500/월)**: 공식 문서에 해당 수치가 **존재하지 않음**을 다중 출처로 확인 → "추정치/범위"로 완화.
- **시점 변동 수치(RunPod $1.34, KLRI 1,559/1,298)**: 2026.6 기준 공식 페이지로 현행값 확인 → 갱신.
- **매출 79억**: 혁신의숲·감사보고서 기반 독립자료가 2022년 약 9억(8.8억)·영업손실 25.2억으로 수렴, theBell 추이와 정합 → 79억 반증, 정정.

3차 결과 모든 항목이 confirmed 또는 근거 있는 correction으로 수렴 → **미확정(uncertain) 0건**.

## 4. ④ 결론

1. 문서 01~05의 **뼈대(코퍼스 통계·구조·아키텍처 권고)는 견고**하며 실측·1차 출처로 뒷받침된다.
2. 발견된 18개 오류는 **전부 정정 가능**한 유형(합산·귀속·추정치·시점)으로, 근거를 찾아 수정했고 **삭제(순수 할루시네이션)는 0건**.
3. 정정의 대부분은 "**과장·단정을 정확한 범위·범주로 낮춘 것**"이다(예: 법률 SOTA → 범용 SOTA, 점추정 가격 → 범위). 결론(스택 권고·배포 경로·데이터 갭)은 유지된다.
4. 본 검증으로 문서 신뢰도가 실무 인용 가능한 수준으로 올라갔다. 단, **시점 민감 수치(가격·시세·서비스 규모)는 인용 시 기준일 명시·재확인 권장**.
