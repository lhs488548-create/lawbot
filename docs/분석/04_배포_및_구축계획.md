#### A. 자유법봇 — 비용 최강, 확장성·견고성 취약


### 2.4 점수 해석

- **B(lawbot-kr)가 34.3점으로 종합 1위**: 세 관점 모두에서 안정적(35/33/35). 특히 **실현가능성(8)·유지보수성(8)**이 높아 0→1 출발점으로 최적.

## 3. 권장 배포 경로 (초보 → 프로덕션, 점수 근거 확정)

세 제안의 강점 영역(B의 출시속도, A의 비용효율, C의 확장성)을 **트래픽·예산 성장에 맞춰 단계적으로 전환**한다. 이는 verdicts가 일관되게 권고한 "손익분기 기반 이관" 전략과 일치한다.

```
[Phase 0] PoC / 학습           → 로컬 Docker Compose (A 스택 축소판, 무료)
        │  목표: 검색 한 줄 띄우기. 노트북에서 1만 청크로 end-to-end 검증
        ▼
[Phase 1] MVP / 베타 (0→1)     → 매니지드 (B: lawbot-kr) ★출발점 확정★
        │  목표: 1~2주 내 공개 URL 출시. 운영부담 0. 트래픽 검증
        │  근거: 종합 1위(34.3), 실현가능성 8, 유지보수성 8
        ▼
[Phase 2] 성장 / 비용 최적화   → 자체호스팅 (A: 자유법봇)
        │  트리거: 월 LLM/임베딩 API 비용 > $200 또는 일 1,000+ 쿼리
        │  목표: 임베딩·검색·재랭킹 로컬화로 변동비 절감. GPU 1장 도입
        │  근거: 비용 8~9. 단 16GB+ 노드·증분갱신·HA 보강 필수
        ▼
[Phase 3] 프로덕션 (1→N)       → 하이브리드 K8s (C: LawBot-KR)
           트리거: 일 1만+ 쿼리, SLA 요구, 무중단 배포 필요
           목표: HPA 자동 스케일·관측성·CI-CD·HA. 검색 한계비용 0 수렴
           근거: 확장성 7~9. 단 변호사법 가드레일·SPOF 해소 추가
```

### 단계 전환 손익분기 (verdicts 종합)

| 전환 | 트리거 지표 | 핵심 동기 |
|---|---|---|
| Phase 1 → 2 | LLM API 월 비용 > $200, 또는 임베딩 재인덱싱 빈번 | 변동비를 고정비(GPU/VPS)로 전환해 단가 하락 |

### 4.2 요청/응답 스키마 (Pydantic)

```python
# 요청
class AskRequest(BaseModel):
    query: str = Field(..., min_length=2, max_length=2000)
    filters: Filters | None = None        # 지역·종류·시점 사전 필터
    stream: bool = True                    # SSE 여부
    citation_mode: Literal["fast", "accurate"] = "accurate"  # 법률은 accurate 권장
    top_k: int = Field(8, ge=1, le=20)

class Filters(BaseModel):
    sido: list[str] | None = None          # ["광주광역시", "전라남도"]
    ord_kind: list[str] | None = None      # ["조례", "규칙", "고시"]
    doc_type: list[str] | None = None      # ["판례", "자치법규", "행정규칙"]
    court_level: list[str] | None = None   # ["대법원", "하급심"]
    case_category: list[str] | None = None # ["민사", "형사", "일반행정"]
    effective_on: date | None = None       # 이 시점에 유효한 조문만

# 인용 객체 (Anthropic Citations / Cohere RAG citations 모델 차용)
class Citation(BaseModel):
    cited_text: str                        # 인용 원문 구절
    source_id: str                         # alr_bdt_id 또는 판례일련번호
    document_title: str                    # 법령명 또는 사건명
    location: str                          # "제12조" 또는 "선고 80도1448"
    source_url: HttpUrl                     # law.go.kr / elis.go.kr
    effective_from: date | None = None     # 시행일 (현행성 표시)
    trust_grade: Literal["A", "B"] | None = None

class AskResponse(BaseModel):
    answer: str
    citations: list[Citation]
    disclaimer: str                        # 법률자문 아님 면책고지 (자동 삽입)
    is_ai_generated: Literal[True] = True  # AI기본법 표시 의무
    cutoff_note: str                       # 데이터 기준일 / 최신 개정 미반영 경고
```

### 4.3 인증 (Authentication)

phase1 권장: **단계화된 인증**.

| 티어 | 인증 방식 | 허용 범위 | 레이트리밋 |
|---|---|---|---|
| **익명(공개 데이터)** | 없음 | `/v1/search`, `/v1/statutes/*` 읽기 전용 | IP당 60 req/h (낮음) |
| **API key** | `X-API-Key` 헤더 | 위 + 상향 검색 | key당 1,000 req/day |
| **Bearer / JWT** | `Authorization: Bearer <jwt>` | `/v1/ask` (LLM 비용 발생) | 토큰 기반 종량제 또는 좌석제 |
| **엔터프라이즈** | OAuth 2.0 + RBAC + 감사로그 | 전체 + SSO | 펌 단위 throttling |

- 근거: 데이터가 공공저작물(§7)이므로 **읽기 전용 검색은 익명 허용**(국가법령 OpenAPI의 개발계정 10,000 req 모델 참고), **LLM 답변만 인증 분리**.
- JWT는 stateless. 비밀키는 환경변수/Secrets로 관리, 코드 미포함.

### 4.4 스트리밍 (Streaming)

phase1 권장: **SSE(Server-Sent Events)**. Cohere/Anthropic 표준 차용.

```python
from fastapi.responses import StreamingResponse

@app.post("/v1/ask")
async def ask(req: AskRequest):
    async def event_stream():
        # 1) 검색·리랭크 (논스트리밍)
        chunks = await retrieve_and_rerank(req)
        # 2) 인용은 별도 이벤트로 먼저/나중 전송 (accurate 모드 = 완성 후)
        # 3) LLM 답변 토큰 스트리밍
        async for token in llm.astream(prompt):
            yield f"event: token\ndata: {json.dumps({'text': token})}\n\n"
        # 4) citation-start 이벤트로 구조화 인용 전송
        yield f"event: citations\ndata: {json.dumps(citations)}\n\n"
        # 5) 면책고지 + AI생성표시
        yield f"event: meta\ndata: {json.dumps(meta)}\n\n"
        yield "event: done\ndata: {}\n\n"
    return StreamingResponse(event_stream(), media_type="text/event-stream")
```

- **fast vs accurate 모드**: 법률 답변은 인덱스 정렬 정확도가 중요하므로 **accurate(완성 후 인용) 기본값**.
- **Nginx/인그레스에서 `proxy_buffering off`** 필수 (SSE 버퍼링 방지).

### 4.5 레이트리밋 (Rate Limit)

phase3 권장: **slowapi + Redis(분산)**.

```python
from slowapi import Limiter
from slowapi.util import get_remote_address

# IP가 아닌 API key / JWT claim 기준 (공유 IP 환경 대응)
limiter = Limiter(key_func=lambda req: req.headers.get("X-API-Key") or get_remote_address(req),
                  storage_uri="redis://redis:6379")

@app.post("/v1/ask")
@limiter.limit("30/minute;500/day")   # LLM 비용 엔드포인트는 엄격
async def ask(...): ...
```

- Redis(또는 서버리스 Upstash)로 분산 토큰버킷. **LLM 비용 엔드포인트는 엄격, 검색은 관대**.
- **시맨틱 캐시**(동일 질의 24h 캐시)로 LLM·리랭크 비용 절감.

### 4.6 RAG 검색·생성 코어

phase3 종합 권장 설계:

| 단계 | 구성 | 비고 |
|---|---|---|
| 임베딩 | **KURE-v1**(bge-m3 파인튜닝, MIT, 1024d, 8192토큰, MTEB-ko 1위) | 범용 한국어 검색 SOTA(법률 포함 다도메인) |
| 검색 | **하이브리드** = dense(KURE) + BM25 sparse(**Nori/kiwi 형태소**) | 조문번호·법령명 정확매칭 + 의미검색 |
| 융합 | **RRF**(Reciprocal Rank Fusion), top-50 후보 | 가중 alpha 0.7~0.8 |
| 리랭크 | **dragonkue/bge-reranker-v2-m3-ko**(크로스인코더, GPU 50~100ms) | top-50 → top-8 |
| 메타필터 | Qdrant payload 사전필터(region/ord_kind/effective_from/trust_grade) | 시점·지역 정밀 검색 |
| 생성 | Claude Haiku/Sonnet 또는 4o-mini(Phase별), grounded only | 인용 강제 + 면책 자동삽입 |
| 검증 | 사후 인용검증(조문번호 정규식 → alr_bdt_id 대조) | 환각·가짜인용 차단 |

> **환각 억제 필수**: Stanford 연구(JELS 2025)는 RAG 기반 상용 법률 도구조차 **17~33% 환각**을 보였다(Lexis+ 17%, Westlaw 33%, GPT-4 43%). "RAG=환각 해결"로 전제하지 말고 **인용검증 레이어 + 환각률 KPI 추적** 필수.

### 7.1 변호사법 §109

