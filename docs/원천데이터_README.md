## 중복 정리 내역 (왜 이렇게 남겼나)

기존에 같은 데이터가 **두 형태**로 있었음 → 정본인 클론만 여기에 모으고, 부분집합인 zip은 제외.

| 데이터 | 정본(여기 채택) | 중복(제외됨) | 사유 |
|---|---|---|---|
| 국가법령 | `01_국가법령` (클론) | (없음) | 유일 |
| 자치법규 | `02_자치법규` (18개 시도 클론) | `법률데이터/statutes-{광주,세종,전남,전북}.zip` (4개 시도) | 클론이 **상위집합** |
| 행정규칙 | `03_행정규칙` (클론) | `법률데이터/statutes-행정규칙.zip` | 동일 원천 |
| 판례 | `04_판례` (클론) | `법률데이터/cases-판례.zip` | 동일 원천(커밋수 일치 확인) |

- 원래 위치 `../data/{legalize,ordinance,admrule,precedent}-kr` → **여기로 이동(move)** 함. `../data/`에는 클론 로그만 남음.
- `../법률데이터/*.zip` 6개는 **전부 위 클론의 부분집합**이라 빌드에 불필요(삭제 시 약 9.5GB 회수 가능 — 사용자 확인 후 삭제 예정).

## 인덱싱 단위 (빌드 시)

- 법령/자치법규/행정규칙: **조문 1개 = 청크 1개**
- 판례: **섹션 1개 = 청크 1개**(긴 섹션은 재분할)
- 과거버전은 제외, **현행만** 인덱싱.

## 주의

- 별표·별지는 현재 **본문이 아니라 라벨/링크**만 있는 경우 많음(메타로 보존).
- 행정규칙 일부는 본문 결측(메타만) → `trust_grade=B`로 분리 표기.
- 빠진 데이터(있으면 좋음, MVP엔 불필요): 헌재 결정례·법령해석례·행정심판 재결례.
